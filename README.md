# 🎬 Netflix Analytics — Executive Content Strategy Project

An end-to-end data analytics project that turns the Netflix Titles catalog
into executive-ready insight: where content comes from, what's growing,
who the recurring talent is, and where to invest next. Built to simulate
the real workflow of a Data Analyst on Netflix's Content Strategy team.

**Stack:** Python · SQLite (SQL) · Pandas · Scikit-learn · Streamlit · Plotly · ReportLab

## 🚀 Quick Start

```bash
pip install -r requirements.txt

# 1. Clean the raw data → produces data/netflix_titles_clean.csv + bridge_*.csv
python src/data_cleaning.py

# 2. Load into SQLite → produces data/netflix.db with views
python src/db_utils.py

# 3. Validate the SQL query library (optional)
python src/run_all_queries.py

# 4. Launch the interactive dashboard
streamlit run dashboard/app.py

# 5. Generate the PDF executive report
python reports/generate_pdf_report.py
```

To re-run the notebooks from scratch:
```bash
jupyter nbconvert --to notebook --execute --inplace notebooks/*.ipynb
```

## 📊 Dataset

**Source:** Netflix Titles dataset (public catalog metadata), 7,787 raw rows
mirrored via the `rfordatascience/tidytuesday` GitHub repository.

| Field | Description |
|---|---|
| `show_id`, `title`, `type` | Identity (`Movie` / `TV Show`) |
| `director`, `cast`, `country` | Comma-separated, multi-valued |
| `date_added`, `release_year` | When added to Netflix vs. originally released |
| `rating`, `duration`, `listed_in` | Maturity rating, runtime, genre tags |
| `description` | Synopsis text (used by the ML recommender) |

**Known limitation:** there is no IMDb score/rating column in this dataset —
`rating` is a *maturity certification* (PG-13, TV-MA, etc.), not a quality
score. This is why "predict IMDb rating" is explicitly scoped out in Phase 8
rather than faked — see `notebooks/04_machine_learning.ipynb`.

## 🧹 Data Cleaning Methodology (Phase 2)

| Step | Approach |
|---|---|
| Missing values | `director`/`cast`/`country` → `"Unknown"` placeholder; `rating` → `"Not Rated"`; `date_added` left null and **flagged**, never imputed |
| Duplicates | Exact row dedup + fuzzy dedup on `(title, type, release_year)` |
| Date standardization | Free-text dates parsed to ISO datetime + year/month/quarter/weekday helper columns |
| Duration parsing | Split into `duration_minutes` (movies) and `duration_seasons` (TV) — never coerced into one misleading number |
| Category standardization | `UR`→`NR` rating merge; derived `maturity_level` bucket (Kids/Family/Teens/Adults/Unrated) |
| Multi-valued fields | Exploded into 4 long-format **bridge tables** so SQL JOINs/GROUP BYs attribute a title to *all* its countries/genres/cast, not just the first one |

Full before/after evidence is in `notebooks/01_data_cleaning.ipynb`.

## 🗃️ SQL Layer (Phase 4)

`sql/02_queries.sql` contains **50 queries**, organized by technique:

| Section | Techniques | Query count |
|---|---|---|
| A | Aggregate functions & GROUP BY | 10 |
| B | JOINs (inner, self, multi-table) | 5 |
| C | CTEs | 5 |
| D | Window functions & ranking (RANK, ROW_NUMBER, NTILE, LAG, FIRST_VALUE) | 8 |
| E | CASE statements | 5 |
| F | Subqueries (scalar, correlated, in WHERE) | 5 |
| G | Date functions | 5 |
| H | Views | 2 |
| I | Business/growth queries | 5 |

Every query was executed against `data/netflix.db` and validated —
`reports/query_results.md` has the actual output of all 50. Run
`python src/run_all_queries.py` to regenerate it yourself.

7 reusable **views** (`sql/03_views.sql`) pre-aggregate country/genre/director/
actor/rating summaries — these are what the dashboard and Power BI guide
query directly instead of re-deriving aggregates everywhere.

**Dialect note:** Written for SQLite; every query is portable ANSI-ish SQL.
The only PostgreSQL/MySQL tweaks needed: `AUTOINCREMENT`→`SERIAL`/`IDENTITY`,
and SQLite's `julianday()`/`strftime()` date functions have direct equivalents
in `EXTRACT()`/`DATE_PART()` on Postgres.

## 📈 Interactive Dashboard (Phase 6)

**Power BI Desktop is a Windows/Mac GUI app and can't run in this Linux build
environment**, so the working, clickable Phase 6 deliverable is a Streamlit
app (`dashboard/app.py`) covering the exact same ground:

1. 🏠 Executive Overview — KPI cards + dynamic title
2. 🌍 Global Content Map — Plotly choropleth
3. 🎭 Genre Analysis — treemap, growth chart, diversification opportunities
4. 🌐 Country Performance — **click-to-drill-through** by country
5. 📈 Release Trend Dashboard — monthly trend, YoY growth, seasonality
6. ⭐ Ratings Dashboard — maturity mix, rating-by-country heatmap
7. 🎞️ Movie vs TV Dashboard
8. 🔍 Title Search & Detail — search any title + "because you watched X" recommendations
9. 🏆 Director & Actor Rankings — with drill-through detail lookup
10. 📅 Netflix Expansion Timeline — first-title-added per country
11. 📌 Executive Summary & Recommendations

Global filters (Year, Country, Genre, Rating, Content Type) live in the
sidebar and apply across every page, with a dynamic filter-summary caption.
Validated with a headless test suite (`dashboard/test_dashboard.py`) using
Streamlit's official `AppTest` framework — all 11 pages pass with zero
runtime exceptions.

**If you have Power BI Desktop**, `dashboard/powerbi_guide.md` has a
page-by-page rebuild guide with DAX measures. **To deploy this dashboard
online**, see `dashboard/DEPLOYMENT.md` (Streamlit Community Cloud, Hugging
Face Spaces, or Docker).

## 💡 Key Business Insights (Phase 7)

- **United States (3,297 titles, ~42%) and India (990, ~13%)** dominate the
  catalog; India's footprint is almost entirely International Movies.
- **TV Show production accelerated sharply after 2016** while movie output
  plateaued — consistent with the platform's strategic pivot to serialized
  originals for retention.
- **Niche genres** (Anime, Faith & Spirituality, Classic/Cult Movies, Sports)
  are present across many markets but have low absolute title counts —
  durable, under-served demand and a clear diversification lever.
- **Rating mix differs meaningfully by country** — useful for localized
  content-maturity strategy and parental-control messaging.
- **Release timing shows real seasonality**, useful for content-drop
  calendar planning relative to competitor activity.

Full narrative + recommendations: dashboard page 11, or
`reports/Netflix_Executive_Report.pdf`.
